My Cozy Game Project

One of my hobbies is playing video games, so I decided to try building one. This page is my month-one roadmap for a cozy, Animal-Crossing-style game prototype made in the Godot engine. The goal is simple: after about four weeks of leftover-time work, have a little character walking around an island, gathering items, fishing, with a real-time day/night cycle — then decide, with real information, whether this becomes a long-term project.

The goal: a playable prototype with placeholder art in ~4 weeks (15–20 total hours).
Engine: Godot 4.7 stable, standard GDScript build, Mobile renderer.

Week 0 — Setup (~2 hrs)

  • Download Godot 4.7 stable (macOS universal build).
  • New 2D project, Mobile renderer, project name cozy-proto.
  • Set a phone resolution (e.g. 1170×2532), stretch mode canvas_items, aspect keep — doing this on day one avoids a painful retrofit.
  • Install export templates that match 4.7 exactly.
  • Set up Git immediately and ignore the .godot/ folder — scene files are text and diff well.

Not yet: iOS export, IAP plugins, App Store anything. That is month 3+ territory.

Week 1 — Learn the engine (4–5 hrs)

The mental model: everything is a Node, and nodes compose into Scenes (a scene is like a reusable component with its own script). Signals are events — a fishing bobber emits fish_bit, the UI listens. GDScript is Python-flavored and reads fluently in about an hour.

  • Complete the official "Your first 2D game" tutorial (the dodge-the-creeps game). ~2 hrs. Teaches nodes, signals, scenes, input, and collision.
  • Skim the docs on TileMapLayer, AnimatedSprite2D, Area2D, and Autoloads (singletons for the game clock and save data).

Skip: 3D, shaders, multiplayer, C#, and any tutorial series longer than 30 minutes per topic. One tutorial, then build.

Week 2 — The island and the walk (4–5 hrs)

Grab placeholder art first: Sprout Lands by Cup Nooble (itch.io) is the de-facto standard cozy asset pack. Don't commission or make art — asset pack, move on.

  • One small island map using TileMapLayer.
  • Player scene: CharacterBody2D + AnimatedSprite2D + CollisionShape2D, ~20 lines of movement code.
  • Camera2D following the player with smoothing on.
  • Keyboard input for now; virtual joystick can wait.

Done when: you can walk a little character around an island and it feels pleasant.

Week 3 — Gathering and the game clock (4–5 hrs)

  • Pickups: an Item scene (Area2D + Sprite2D); on body_entered add to inventory and free the node.
  • Inventory: a GameState.gd Autoload holding item counts, shown in a simple panel. Ugly is fine.
  • The clock (the soul of the genre): a GameClock.gd Autoload advancing an in-game hour every real minute, with a CanvasModulate tinting the screen by time of day. The moment day fades to night is when it starts feeling like a game.
  • Respawning: items come back the next in-game day, so there's a reason to return.

Week 4 — Fishing minigame + verdict (4–5 hrs)

  • Water tiles near shore become fishable Area2D zones.
  • Cast → bobber → random 2–6s wait → bobber shakes → short reaction window → catch or miss. Catches use a weighted rarity table.
  • A tiny "You caught a bass!" popup with the sprite.
  • Save/load: serialize GameState to a local JSON file with FileAccess on quit, load on start.

Then the verdict. Was building this fun, or did it feel like a second job? Did I open the project just to tinker, unprompted? Did the rest of life suffer this month? Three good answers → commit to the long roadmap. Any bad answer → shelve it guilt-free — I learned a game engine for the price of one month's leftover hours.

If it's a go: the shape of v1 (months 2–18)

  • Months 2–4: villager system, daily dialogue, friendship points, a museum/catalog UI, bigger island.
  • Months 5–7: grid-snap furniture placement, a shop and currency loop.
  • Months 8–10: seasons, more villagers, sound and music.
  • Months 11–14: real art pass — commission a pixel artist once it's proven fun. Polish and onboarding.
  • Months 15–18: iOS export pipeline, StoreKit 2 for cosmetic IAP, TestFlight beta, App Store launch.
Standing rules
  • This project gets leftover hours only.
  • No new systems until the current one is fun.
  • Every month, the game must be playable start to finish.
